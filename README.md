## 项目经验

### 项目一：电商秒杀系统重构（2022.03 – 2022.08）
**背景**：原有秒杀系统在峰值时出现超卖、页面卡顿，用户体验差。  
**职责**：架构师 & 核心开发  
- **前端层**：CDN静态资源加速 + 浏览器本地限流。  
- **接入层**：Nginx+Lua进行IP/用户粒度的限流，拦截恶意请求。  
- **业务层**：采用“分层过滤”策略，预减库存（Redis）→ 校验资格 → 异步下单（RocketMQ）→ 数据库最终扣减。  
- **数据库层**：库存扣减使用数据库行锁 + 乐观锁，保证不超卖。  
**成效**：系统抗住10万+并发，零超卖，订单处理能力提升10倍。  

### 项目二：全渠道库存中台（2021.01 – 2021.12）
**背景**：多销售渠道（APP、小程序、线下门店）库存割裂，导致超卖或缺货。  
**职责**：技术负责人  
- 设计**库存中心模型**：物理库存、可售库存、预占库存分层管理。  
- 引入**分布式锁与Redlock**保证多节点库存扣减原子性。  
- 通过**Binlog+MQ**实现库存变更的准实时同步到渠道端。  
- 建立**库存健康大盘**：实时监控库存水位，预警超卖风险。  
**成效**：库存准确性提升至99.99%，库存周转率提高20%。  

### 项目三：全链路压测平台建设（2021.05 – 2021.10）
**背景**：大促前无法准确评估系统容量，依赖人工估算，风险高。  
**职责**：核心贡献者  
- 调研并引入**流量录制与回放**工具（GoReplay、JVM-Sandbox），生产流量录制后脱敏压测。  
- 设计**影子表方案**：压测流量携带标记，数据隔离写入影子库/表，不影响线上。
